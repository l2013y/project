会展及论坛总结&及后续项目发展

2026-05-30
#会展论坛

2026.5.30 论坛参与流程

今天下午抵达 OPC 极客夜话论坛，本次练兵场目标是：用 AI 辅助完成一个校园跑腿小程序的前后端开发。整体流程主要包括模型接入、微信小程序配置、云开发部署、前端联调和问题排查。

  • 论坛目标： 用 AI 辅助完成校园跑腿小程序的前后端开发。
  • AI 工具配置： 配置万界方舟账号与 DeepSeek V4 Pro 模型，并在 Trae IDE 中完成模型接入。
  • 小程序准备： 下载并配置微信开发者工具，注册小程序账号，获取 AppID。

云开发配置与部署

  1. 在微信开发者工具中点击“云开发”按钮，开通云开发服务，创建新环境（例如 dev），获取环境 ID。
  2. 在 app.js 中通过 wx.cloud.init({ env: '你的环境ID' }) 初始化云能力。
  3. 在项目根目录创建 cloudfunctions 文件夹，并在 project.config.json 中配置 cloudfunctionRoot 路径。
  4. 右键 cloudfunctions 文件夹，选择“选择云环境”，绑定刚刚创建的环境。
  5. 在 cloudfunctions 下新建云函数，例如 getOpenid、publishOrder、acceptOrder、updateOrderStatus 等，并编写函数逻辑。
  6. 右键每个云函数文件夹，选择“创建并部署：云端安装依赖”，将函数上传到云端。
  7. 在云开发控制台的数据库中新建集合，集合名必须与代码中查询的名称完全一致，例如 orders、users、addresses。
  8. 根据需要设置数据库权限：跨用户查询（如帮拿员看订单）不能在前端直查，必须改用云函数。

小程序前端开发与联调

  • 基础页面： 完成用户端与帮拿员端的基础页面，包括身份选择、下单、订单大厅、我的订单等。
  • 云函数调用： 通过 wx.cloud.callFunction 调用云函数，避免前端直接查询数据库。
  • 登录流程设计： 首次启动进入全屏登录页，提供“微信一键登录”和“暂不登录”选项；一键登录触发 getOpenid 并缓存登录状态与身份；暂不登录时可以浏览身份选择页，但点击功能入口时需要拦截并提示登录；后续启动根据缓存直接进入对应首页。
  • 真机调试： 遇到主包超限（2.01MB），后续可以通过将图片移入云存储、压缩图片或启用分包加载解决。

今日主要问题与解决方案

1. 帮拿员端看不到订单

问题：云数据库默认权限为“仅创建者可读写”，普通帮拿员无法通过 db.collection('orders').get() 直接查询其他用户的订单。

解决：新建 getOrders 云函数，在云函数中统一查询订单；前端三个页面（帮拿员大厅、帮拿员订单、用户订单）都改用该云函数获取订单数据。

2. 数据库集合名称不一致

问题：代码中使用的是 orders，但控制台中创建的是 order，导致查询不到对应数据。

解决：统一集合名称，推荐使用复数形式，并确保大小写完全一致。

3. 真机调试主包超限

问题：代码包大小超过 2MB，导致真机调试失败。

解决：将图片上传到云存储，或压缩图片，必要时启用分包加载。

云开发部署要点总结

  • 云函数修改后，必须重新右键选择“上传并部署：云端安装依赖”。
  • 前端代码修改后只需要重新编译，不需要重新部署云函数。
  • 数据库集合名必须与代码完全一致，并且大小写敏感。
  • 跨用户数据操作必须通过云函数，不能依赖前端直查。
  • 图片等静态资源优先使用云存储，避免主包超限。

今日收尾状态

  • 用户端与帮拿员端基础流程已经可以跑通，包括下单、接单、订单流转。
  • 帮拿员端看不到订单的问题原因已定位，修复方案已确定为新增 getOrders 云函数。
  • 登录流程逻辑已设计完成，待后续开发实现。
  • 云开发环境、数据库、云函数均已部署并验证可用。

目前仍存在一个问题：小程序端点击“微信登录”后再次点击主页按钮又会回到登录页面，但点击“暂不登录”可以看到主页面，并且能够正常使用。当前目标是让未登录用户无法使用项目功能，但目前效果更像是只做了“未登录提示”，后续需要继续完善登录拦截和页面跳转逻辑。后续功能还待调整。

2026.6.10 再三思考之后还是决定更换项目方向

了解到小程序一旦涉及金额交易，上线门槛就会比较高。不过不涉及金额交易的小程序，可以通过后续广告接入来收取广告费，所以项目目前改版为校园墙小程序。

1. 广场页面

  • 帖子列表展示，支持“最新”和“最热”排序。
  • 支持下拉刷新、上拉加载更多。
  • 点击帖子可进入详情页。

2. 帖子卡片组件

  • 展示头像、昵称、时间、正文和图片九宫格。
  • 显示点赞数和评论数。
  • 支持点赞 / 取消点赞，并带有动画效果。

3. 发帖功能

  • 支持文本输入，最多上传 9 张图片。
  • 支持匿名发布。
  • 发布后自动返回广场并刷新列表。

4. 帖子详情页

  • 查看完整内容，支持图片放大预览。
  • 支持点赞和发表评论。
  • 评论列表支持分页展示。

5. 我的页面

  • 支持微信授权登录，并同步用户信息。
  • 支持查看我的帖子列表，复用帖子卡片组件。
  • 我的收藏和设置目前为占位入口。

6. 数据存储

  • 使用云开发，包含 users、posts、comments、likes 四个集合。
  • 帖子、评论、点赞均关联用户身份。
  • 匿名帖子前台展示匿名标识，后台仍可追溯真实用户。

7. 云函数

  • getPosts：获取帖子列表 / 详情。
  • publishPost：发布帖子。
  • likePost：点赞 / 取消点赞。
  • getComments：获取评论列表。
  • addComment：发表评论。
  • upsertUser：同步用户资料。
  • migrateLegacyLikes：迁移旧点赞数据。

8. 工具函数

  • formatTime：时间格式化。
  • uploadImage：图片上传至云存储。
  • normalizePost：统一帖子数据格式。

当前状态： 小程序已经完成核心闭环，用户可以浏览帖子、发布内容，并进行评论和点赞互动。

2026.6.11 对校园墙小程序的进一步改动

  • 完善通知红点逻辑：进入“我的帖子”看新增，离开后清零，后续从 0 重新计数。
  • 云函数改名更清晰：如 getUnread、markRead、createTestNotice。
  • package.json 描述改成中文。
  • 爱心图标换成 SVG，真机显示更自然。
  • 新增删除自己帖子的功能：
    • “我的帖子”可删除
    • 详情页自己的帖子可删除
    • 别人的帖子不显示删除
  • 详情页新增三点菜单，当前展开项是“评论”。
  • 广场大厅恢复普通展示，不显示详情页专用操作。

2026.6.12 分类、评论与界面细节调整

1. 广场分类功能
把原来的 最新 / 最热 改成了分类筛选：全部 / 失物招领 / 求助。

现在广场右下角有一个固定分类按钮，点击后分类菜单从下往上展开。相关逻辑：category: this.data.activeCategory。云函数 getPosts 根据 category 筛选帖子。如果当前分类没有帖子，会显示：这个分类目前还没有帖子，快来发布吧

2. 发布页增加分类
发布帖子时可以选择：普通 / 失物招领 / 求助。

新帖子会写入字段：category: 'normal' | 'lost_found' | 'help'。普通帖子不显示标签，失物招领和求助会显示标签。

3. 帖子卡片分类标签
帖子卡片会显示类似 失物招领、求助。对应代码里生成了：categoryLabel。普通帖子不显示分类标签，避免首页太乱。

4. 下拉刷新提示
广场下拉刷新后会显示 刷新成功 或 刷新失败，不再只是默默刷新。

5. 评论“楼主”徽章
详情页评论区，如果楼主本人评论，会在昵称旁显示 楼主。判断逻辑是：评论用户 ID === 帖子作者 ID

6. 删除评论功能
新增了云函数：deleteComment
功能：

  • 只能删除自己的评论；
  • 删除后 comments 集合里对应评论会被删；
  • 帖子的评论数会 -1；
  • 对应评论通知也会删除。

前端交互改成：长按自己的评论 → 底部弹出操作面板 → 删除 / 取消。行内“删除”文字已经去掉了。

7. 详情页三点菜单调整
详情页帖子右下角三点菜单里原来的 评论 改成了 收藏。

目前收藏功能还没真正开发，只是预留了入口，点击会提示：收藏功能开发中

8. UI 微调
今天还做了一些界面细节：

  • 详情页评论发送按钮变小，让输入框占更多空间；
  • 底部 tab 中间加号按钮稍微下移；
  • 广场分类按钮固定到右下角；
  • 分类菜单改成向上展开；
  • 评论长按操作面板从底部向上弹出。

2026.6.15 收藏、搜索、资料与设置完善

1. 收藏功能完善
本次补齐了收藏功能的完整闭环：帖子详情页 → 点击收藏 → 写入收藏记录 → 我的收藏展示

主要实现内容：

  • 新增收藏 / 取消收藏逻辑；
  • 帖子详情页支持收藏和取消收藏；
  • 帖子加载时返回当前用户是否已收藏；
  • “我的”页面新增“我的收藏”入口；
  • “我的收藏”页面复用已有帖子列表页面；
  • 收藏为空时只显示文字提示，不显示多余占位符号。

涉及模块：

  • 云函数：收藏 / 获取帖子
  • 页面：帖子详情页、我的页面、我的帖子 / 收藏列表页
  • 组件：帖子卡片组件

这个功能让我第一次比较完整地理解了 前端点击事件 → 云函数 → 数据库 → 页面状态刷新 这一类小程序功能闭环。

2. 广场搜索功能
广场页新增了搜索能力。
支持功能：

  • 输入关键词搜索帖子内容；
  • 搜索结果支持分页；
  • 下拉刷新时保持当前搜索条件；
  • 分类筛选和关键词搜索可以同时生效；
  • 搜索无结果时显示“没有找到相关帖子”；
  • 新增“返回广场”按钮，点击后清空搜索并恢复广场主列表。

这个功能的重点不是单独写一套搜索页面，而是复用了原来的广场帖子加载逻辑，只是在请求参数里增加了关键词条件。这样做的好处是：

  • 代码更精简
  • 分页和刷新逻辑不用重复写
  • 后续维护成本更低

3. 编辑个人资料功能
“我的”页面顶部用户信息区域支持点击进入编辑资料页。
支持功能：

  • 修改昵称；
  • 修改头像；
  • 头像上传到云存储；
  • 保存个人资料；
  • 返回“我的”页面后自动刷新头像和昵称。

实现思路：我的页面 → 编辑资料页 → 保存用户信息 → 更新本地缓存 → 返回后刷新展示。

同时，编辑资料入口没有额外保留“编辑”按钮，而是直接让用户点击资料栏进入编辑页面，减少页面视觉负担。

4. 历史帖子和评论资料同步
开发过程中发现一个问题：修改头像和昵称后，之前发布过的帖子仍然显示旧昵称和旧头像。

原因是帖子和评论里保存的是发布当时的昵称和头像副本，而不是每次都动态读取用户表。
解决方式：

  • 保存个人资料时，同步更新当前用户以前发布的非匿名帖子；
  • 同步更新当前用户以前发表的非匿名评论；
  • 匿名帖子和匿名评论不更新，保持匿名逻辑。

这个点让我理解到一个重要概念：冗余字段 / 数据副本。

也就是为了页面展示方便，帖子里会保存一份昵称头像，但这样也会带来资料更新后的同步问题。

5. 设置页功能
新增了设置页第一版。目前包括三个功能：

  • 默认匿名发布
  • 清除缓存
  • 关于校园墙

默认匿名发布
用户可以在设置页开启默认匿名发布。
效果：

  • 开启后，进入发布页时默认勾选匿名；
  • 用户仍然可以在发布页手动关闭；
  • 发布成功后会恢复到用户设置的默认状态。

清除缓存
清除缓存功能经过调整后，只清理临时缓存，不再一键清除用户设置和个人资料。
目前不会清除：

  • 个人资料
  • 默认匿名设置
  • 云端帖子
  • 评论
  • 收藏
  • 通知

这样更符合用户直觉，避免误删用户已经配置好的内容。

关于校园墙
“关于校园墙”从设置页中的静态说明，改成了一个单独页面。
特点：

  • 点击设置页的“关于校园墙”进入；
  • 页面只读；
  • 只保留版本模块；
  • 后续每次版本更新，可以新增一个独立版本模块。

目前版本模块示例：

  • 版本 1.0.0
  • 完成广场、发布、详情、点赞、评论、收藏、搜索、个人资料和基础设置功能。
