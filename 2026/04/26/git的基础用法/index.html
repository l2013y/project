git的基础用法

2026-04-26
#学习

Git 基本流程

Git 日常使用可以简单理解成：先把远程代码拉到本地，在本地修改后提交到本地仓库，最后再推送到远程仓库。

  • git clone：第一次把远程仓库复制到本地。
  • git pull：把远程仓库的最新代码拉到本地。
  • git add：把修改加入暂存区。
  • git commit：把暂存区内容提交到本地仓库。
  • git push：把本地提交推送到远程仓库。

如果别人在远程仓库也改了同一部分代码，执行 git pull 时可能会产生冲突，需要先解决冲突，再继续提交和推送。

分支的基本用法

一般开发时不会直接在主分支上改代码，而是先新建一个功能分支，写完后再合并回主分支。

  • git checkout -b feature-login：新建并切换到 feature-login 分支。
  • git switch 分支名：切换分支，推荐用于单纯切换分支。
  • git checkout 分支名：也可以切换分支。
  • git merge 分支名：把指定分支合并到当前分支。

git checkout 还有恢复文件的作用，比如 git checkout 文件名 会用暂存区或版本库里的文件覆盖工作区文件。它既能切分支，又能恢复文件，所以现在单纯切分支时更推荐用 git switch。

日常工作流程

一个比较常见的工作流程是：

  1. 早上先执行 git pull，同步远程最新代码。
  2. 新建功能分支：git checkout -b feature-login。
  3. 写代码、改文件。
  4. 用 git status 查看当前修改状态。
  5. 用 git add . 暂存修改。
  6. 用 git commit -m "完成登录框样式" 提交到本地。
  7. 功能完成后切回主分支：git checkout main。
  8. 再执行一次 git pull，确保主分支是最新的。
  9. 合并功能分支：git merge feature-login。
  10. 确认没问题后执行 git push。

远程仓库相关命令

  • git remote -v：查看当前连接的远程仓库地址。
  • git remote show origin：查看远程仓库 origin 的详细信息。
  • git remote add origin 远程仓库地址：添加远程仓库地址。
  • git remote remove origin：删除当前名为 origin 的远程仓库连接。

其他常用命令

  • ls -la：查看所有文件，包括隐藏文件。
  • ls：只查看当前目录下的文件和文件夹名称。
  • dir：以 Windows 风格查看目录。
  • git ls-files：查看已经被 Git 管理的文件。
  • git status：查看当前工作区状态。
  • nvm list：列出所有 Node 版本。
  • nvm use 版本号：切换 Node 版本。
  • git push origin 本地分支名:远程分支名：把本地分支推送到指定远程分支。
  • git push --force-with-lease origin 本地分支名:远程分支名：用本地分支覆盖远程分支，比 --force 更安全，但执行前仍要确认远程旧提交可以被覆盖。

储物柜(用来保存未提交到本地的代码但想切换分支）

git stash：暂存所有的未提交改动
git stash push -m “对此次保存的描述”: 给暂存添加备注（目的是为了方便查找）
git stash list: 查看所有的暂存列表
git stash pop: 应用最近一次暂存，并删除它
git stash apply: 应用最近一次暂存，但不删除（可重复应用）
git stash drop: 删除最近一次暂存
git stash clear: 清空所有暂存
如果有多个stash暂存的话就用git stash list查看所有的暂存列表：
示例：
stash@{0}: On dev2026: 骑手接单页面
stash@{1}: On master: 修复登录bug
stash@{2}: On dev2026: 数据库配置
想应用哪个的话就用git stash apply/pop ‘stash@{对应的索引号}’

Tips

一般来说，只有新增、删除、修改过的代码或文件，才需要执行 git add、git commit 等操作。
