2026.4.27

刚面对新项目

对于这个项目毫无头绪，甚至哪个文件夹是干嘛的都不知道，只记得 npm install 和 npm run dev 这两个命令。
2026.4.29

1. 前端（vecode）

在借助AI工具的帮助下，在shoptour-backstage、shoptour-frontend两个前端项目中分别运行了 npm install 和 npm run dev 可以正常启动，端口号为 http://localhsot:8080 。

2. 后端（IDEA）

在进入上述端口之后是一个登录页面，这时候需要进去idea找到shoptour-platform中的web文件夹，运行三个包中的agent和app文件夹中的jar包，这时候就可以在之前的8080端口输入 http://localhsot:9088/admin/doc.html ，之后再终端输入 Test-NetConnection -ComputerName localhost -Port 9088 测试连通性，终端输出true则连通成功。

如果运行app文件中的jar包，之后输入http://localhost:9090/app/doc.html。

运行AgentRun之后会自动退出，我让通义灵码查看之后给出结果是没有引入spring-boot-starter-web依赖，但考虑到不一定需要安装这个依赖，所以并没有添加

2026.5.8

前后端边界

理解了前端的用户权限和后端的数据管理，前端用户一般只有新建账号或者删除自己账号/其他内容，后端只有管理员可以删除数据，处理垃圾信息或违规内容。

暴力新建账户

注册页面一般会有验证码或令牌对新建速率进行限制，还有一种方法是某个接口的流量突然飙升，系统会启动“熔断机制”，这个接口会停止响应，等到攻击结束再恢复。

断点

对含有断点的代码进行Debug调试时，含有断点的那一行代码以及后续的代码都不会被执行

  1. 对第4行打上断点进行调试，结果会在线程和变量面板出现 name = "张三" age = 25
  2. 对第5行打上断点继续进行测试（也可以用F8快捷键对下一行进行断点调试），线程和变量面板会显示 name = "张三" age = 25 result = "你好，张三先生/女士" ，继续向下调试第6行，控制台输入result，继续向下调试，程序结束。（线程和变量面板和控制台差不多，但本质是两个东西，变量面板是看内存里有什么，控制台时看程序输出了什么）
  3. 虽然说方法的代码在断点之后，但是方法被调用在断点之前，所以断点在第五行时线程和变量可以正常显示result
    快捷键：F8，继续向下一行调试；F7，步入断点这行所调用的方法代码内部进行调试

2026.5.9
xjsp-common中result文件夹主要用于用户的登录时返回的响应结果和提示用户是否有访问权限，例如抛出用户密码或者账号是否正确、用户是否存在。
exception文件夹用来处理系统异常，例如某个操作需要特定操作而没有进行特定操作，那么就会抛出“操作异常”，如果异常较多，可以定义全局处理器：比如第一个处理器是处理参数校验失败的，那么这个处理器会自动捕获参数校验异常，统一转换为R.error(一般是报错代码+错误文本)如：（500，”操作异常”),之后前端会收到

全局处理器的存在能够使代码更简洁，处理异常操作更快捷。全局处理器包名一般以Handler结尾
Constant文件夹作用是集中管理项目中各种固定不变的值

2026.5.15
利用claude code对巡店系统的任务派发模块添加了一个‘进度’按钮，其中包括‘待处理、处理中、待验收、任务结束’四个选项，最初的问题是只能在网页上显示，但是页面无法保存，刷新页面又会返回’待处理‘这个默认选项，之后在本地复制了远程数据库的数据之后，点击四个选项的其中一个还是报错，AI分析原因之后是数据库中没有字段，在数据库添加完字段之后再次试验就可以正常运行了。
